addAudioModeChangedListener

Defined in api\G2NMobile.js:190

Syntax

addAudioModeChangedListener

(
  • listener
  • callback
)

Summary

This method use when detect event about change of audio mode from Android-Phone. To detect changed event by audio mode, add the listener.

See Also : removeAudioModeChangedListener

Parameters:

  • listener Function
  • callback Function

Example:

    var audioModeChangedListenerKey = null;

    function showAudioResult(returnValue) {
        $('#result_mobile_audio').text('Result: ' + returnValue[0]);
    }

    function addOnAudioModeChangedListenerCallback(key) {
        audioModeChangedListenerKey = key;
    }

    function removeAudioModeChangedListenerCallback() {
        audioModeChangedListenerKey = null;
    }

    $('#btn_mobile_audio_addAudioModeChangedListener').click(function() {
        G2NMobile.Audio.addAudioModeChangedListener(showAudioResult, addOnAudioModeChangedListenerCallback);
    });

    $('#btn_mobile_audio_removeAudioModeChangedListener').click(function() {
        if ( audioModeChangedListenerKey !== null ) {
            G2NMobile.Audio.removeAudioModeChangedListener(audioModeChangedListenerKey, removeAudioModeChangedListenerCallback);
        }
    });

adjustVolumeDown

Defined in api\G2NMobile.js:394

Syntax

adjustVolumeDown

(
  • volumeType
  • adjustVolumeOption
  • callback
)

Summary

One step down to adjusts the volume of the volume type.

Caution!
If wrong with the inputed value in first parameter for volumeType, you receive the 'G2NAudioVolumeTypeException' message.
If wrong with the inputed value in second parameter for adjustVolumeOption, you receive the 'G2NAudioAdjustVolumeOptionException'.
If audio mode isn't sound mode, you are unobtainable the volume index. It that you receive the 'G2NAudioModeException' message.

See Also : adjustVolumeUp

Parameters:

  • volumeType G2NMobile.Audio.VolumeType

    VOICE_CALL(0), SYSTEM(1), RING(2), MUSIC(3), ALARM(4), NOTIFICATION(5), DTMF(8)

  • adjustVolumeOption G2NMobile.Audio.AdjustVolumeOption

    The way Android-Phone window display the adjusted volume value. SHOW_UI(1), PLAY_SOUND(4)

  • callback Function

Example:

    G2NMobile.Audio.adjustVolumeDown(G2NMobile.Audio.VolumeType.SYSTEM, G2NMobile.Audio.AdjustVolumeOption.SHOW_UI);

getAudioMode

Defined in api\G2NMobile.js:127

Syntax

getAudioMode

(
  • callback
)
G2NMobile.Audio.AudioMode

Summary

Returns the current audio mode of Android-Phone. Audio mode means that sound, vibrate, silent mode of Android-Phone.

See Also : setAudioMode

Parameters:

  • callback Function

Returns:

G2NMobile.Audio.AudioMode:

Returned the NORMAL(2) when the current audio mode of Android-Phone has sound mode. And returned the VIBRATE(1) has vibrate mode. Also, SILENT(0) has silent mode.

Example:

    function showAudioResult(returnValue) {
        $('#result_mobile_audio').text('Result: ' + returnValue);
    }

    $('#btn_mobile_audio_getAudioMode').click(function() {
        G2NMobile.Audio.getAudioMode(showAudioResult);
    });

setAudioMode

Defined in api\G2NMobile.js:158

Syntax

setAudioMode

(
  • audioMode
  • callback
)

Summary

This method is sets the audio mode of Android-Phone. Silent mode will mute the volume and will not vibrate. Vibrate mode will mute the volume and vibrate. Normal mode will be audible and may vibrate according to user settings.

Caution!
If wrong with the inputed value in first parameter for audioMode, you receive the 'G2NAudioModeException' message.

See Also : getAudioMode

Parameters:

  • audioMode G2NMobile.Audio.AudioMode

    When you want to change current audio mode of Android-Phone to sound mode, input the NORMAL(2). To vibrate mode input the VIBRATE(1). To silent mode input the SILENT(0).

  • callback Function

Example:

    G2NMobile.Audio.setAudioMode(G2NMobile.Audio.AudioMode.SILENT);

setVolumeValue

Defined in api\G2NMobile.js:284

Syntax

setVolumeValue

(
  • volumeType
  • volumeValue
  • adjustVolumeOption
  • callback
)

Summary

Set the volume index of the volume type.

Caution!
If wrong with the inputed value in first parameter for volumeType, you receive the 'G2NAudioVolumeTypeException' message.
If not inputed volume value from 0 to max volume value, you receive the 'G2NAudioVolumeValueException'.
If wrong with the inputed value in third parameter for adjustVolumeOption, you receive the 'G2NAudioAdjustVolumeOptionException'.
If audio mode isn't sound mode, you are unobtainable the volume index. It that you receive the 'G2NAudioModeException' message.

See Also : getVolumeValue

Parameters:

  • volumeType G2NMobile.Audio.VolumeType

    VOICE_CALL(0), SYSTEM(1), RING(2), MUSIC(3), ALARM(4), NOTIFICATION(5), DTMF(8)

  • volumeValue Number

    The volume value to set. See getMaxVolumeValue for the largest valid value.

  • adjustVolumeOption G2NMobile.Audio.AdjustVolumeOption

    The way Android-Phone window display the changed volume value. SHOW_UI(1), PLAY_SOUND(4)

  • callback Function

Example:

    G2NMobile.Audio.setVolumeValue(G2NMobile.Audio.VolumeType.SYSTEM, 5, G2NMobile.Audio.AdjustVolumeOption.PLAY_SOUND);
